## 3. 依赖安装结果

### 3.1 首次安装失败（系统 Python）

- 命令：`python3 -m pip install -r requirements.txt`
- 失败原因：
  1. 系统 Python 为 `3.13.9`，与 `numpy==1.26.4` 不兼容；
  2. `requirements.txt` 含 Windows 包：`pywin32==311`、`win32_setctime==1.2.0`。

### 3.2 切换 `.venv` 后安装成功

- Python：`./.venv/bin/python` -> `3.11.9`
- 安装命令（已执行成功）：
  - `iconv -f UTF-16LE -t UTF-8 requirements.txt | sed '1s/^\xEF\xBB\xBF//;/^pywin32==/d;/^win32_setctime==/d' > /tmp/requirements.macos.txt`
  - `./.venv/bin/python -m pip install -r /tmp/requirements.macos.txt`

### 4.2 回归 B：本地最新代码服务（`127.0.0.1:8000`）

### 启动阶段问题

- 首次启动后回归报错：`/auth/send_code` 返回 500。
- 原因：本地 `.env` 默认 `REDIS_URL=redis://127.0.0.1:6379/`，而当前 Redis 映射端口为 `6380`。
- 修复：以临时环境变量重启服务。
  - `REDIS_URL=redis://127.0.0.1:6380/ ./.venv/bin/python server.py`

### 回归结果

- 命令：`./.venv/bin/python scripts/fortune_regression.py --base-url http://127.0.0.1:8000 --timeout 90`
- 结果：
  - `passed=8`
  - `failed=16`
  - `pass_rate=33.33%`

### 失败特征

- 主要失败用例类型：`FORTUNE-002` 到 `FORTUNE-017` 多条。
- 失败原因集中为脚本断言：
  - 旧断言要求固定出现：`命理依据：`、`行动建议：`、`参考置信度：`
  - 新 V2 输出已改为自然骨架（如 `依据：`、`建议：`，且不强制 `参考置信度`）。
- 结论：这 16 条多数为“回归脚本口径与新输出协议不一致”，不等同于接口不可用。
