# 概要设计说明书

- 文档版本：v2.0
- 更新日期：2026-02-27
- 适用范围：当前仓库 `server.py` 主线实现

## 1. 文档目标

本文档用于说明“吉伊大师命理咨询系统”的总体架构、核心模块边界、关键业务流程和非功能要求，作为研发、测试、运维和文档统一基线。

## 2. 系统定位

系统面向命理咨询场景，提供“鉴权登录 + 对话问答 + 命理分析 + 知识检索”一体化能力。

核心能力：

- 鉴权体系：短信验证码登录/注册、账号密码登录、忘记密码
- 智能对话：普通聊天、命理问答、占卜、解梦、星座问答
- 资料管理：自动抽取姓名/生日/时辰/称呼偏好并持久化
- 质量治理：时间一致性校验、质量指标采集、回归门禁
- 知识增强：网页内容向量化并写入本地 Qdrant

## 3. 总体架构

系统采用“单体后端 + 模板前端 + 多存储 + 外部模型服务”架构。

### 3.1 架构分层

- 接入层
  - 浏览器页面：`/index`、`/login`、`/register`、`/forgot-password`
  - API：`/auth/*`、`/chat`、`/quality/metrics`、`/add_urls`
- 应用层（`server.py`）
  - 鉴权与会话管理
  - 资料抽取与融合
  - 多路由问答编排（快捷回复/命理/星座/解梦/占卜/Agent）
  - 输出清洗与时间一致性修补
  - 质量指标统计与可观测性
- AI能力层
  - 主 Agent（LangChain）
  - 命理、占卜、解梦、检索工具（`mytools.py`）
- 数据层
  - MySQL：用户、资料、会话审计、密码重置日志
  - Redis：登录态、验证码、聊天历史、特性开关、质量指标
  - Qdrant（本地）：知识向量库

### 3.2 关键依赖

- LLM/Embedding：阿里百炼 OpenAI 兼容接口
- 搜索：SerpAPI（可选）
- 命理外部能力：缘分居 API

## 4. 模块划分

- 核心后端：`server.py`
- 配置管理：`config.py`
- 模型客户端：`models.py`
- 工具集：`mytools.py`
- 前端脚本：`static/js/main.js`、`auth.js`、`login_multi.js`、`reset_password.js`
- 数据初始化：`sql/001_init_auth_schema.sql`
- 回归与门禁：`scripts/fortune_regression.py`、`scripts/quality_gate.py`

## 5. 核心流程

### 5.1 鉴权流程

1. 客户端调用 `POST /auth/send_code` 发送验证码。
2. 后端校验手机号和冷却时间，验证码写 Redis（300 秒）。
3. 客户端调用 `POST /auth/verify` 登录或注册。
4. 登录成功后写 Redis 登录态、MySQL 审计，并设置 `jiyi_auth_token` Cookie。

### 5.2 聊天流程

1. 校验 Cookie -> Redis 登录态，未登录返回 `401`。
2. 绑定会话 ID 为用户 UUID，构建时间锚点。
3. 从历史和本轮输入抽取资料，合并写入 `user_profile`。
4. 路由优先级：
   - 时间/问候快捷回复
   - 解梦路由
   - 星座路由（缺星座先澄清）
   - 命理/占卜专用路由
   - Agent 回退路由
5. 输出经过清洗、去资料回显、时间一致性校验后返回。
6. 过程指标写入 Redis，用于 `GET /quality/metrics` 聚合展示。

### 5.3 知识入库流程

1. 调用 `POST /add_urls?URL=...`。
2. 抓取网页、切分文本、向量化。
3. 写入本地 Qdrant 集合 `local_documents`。

## 6. 部署架构

- 本地模式：`python server.py`（默认 8000）
- 容器模式（推荐）：`docker compose` 启动 `numerology + mysql + redis`
- 数据初始化：MySQL 启动时自动执行 `sql/001_init_auth_schema.sql`

## 7. 非功能设计

- 可维护性：配置集中在 `.env` + `config.py`
- 可扩展性：通过工具函数和特性开关扩展新路由/新策略
- 可观测性：日志 + 质量指标聚合（`/quality/metrics`）
- 可靠性：外部依赖失败时提供兜底回复，避免接口崩溃

## 8. 约束与风险

- 生产环境必须关闭 `SMS_DEBUG_CODE_ENABLED`。
- 当前 Cookie 未设置 `Secure`，生产 HTTPS 建议开启。
- 命理能力受上游 API 稳定性影响，需保留降级策略。
- 聊天历史默认短期存 Redis，不提供长期审计留痕。
