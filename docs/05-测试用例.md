# 测试用例

## 1. 说明
- 目标：验证当前 MySQL + Redis 真实实现可用
- 范围：验证码鉴权、登录态、用户资料、聊天链路、知识入库
- 前置：
  - 服务已启动
  - Redis 与 MySQL 可连接且表结构可用
  - `.env` 已配置必要密钥

## 2. 鉴权与账户

### TC-AUTH-001 发送验证码成功
- 步骤：`POST /auth/send_code`，合法手机号，`scene=login`
- 期望：`200`，`ok=true`，返回 `ttl_seconds=60`；当 `SMS_DEBUG_CODE_ENABLED=true` 时额外返回 `debug_code`

### TC-AUTH-002 发送验证码失败（手机号非法）
- 步骤：手机号传 `12345`
- 期望：`400`，提示手机号格式错误

### TC-AUTH-003 发送验证码失败（冷却中）
- 前置：同手机号 60 秒内重复发送
- 步骤：连续调用 `/auth/send_code`
- 期望：第二次 `429`，提示剩余冷却秒数

### TC-AUTH-004 注册成功
- 前置：手机号未注册，已拿到验证码
- 步骤：`POST /auth/verify`，`mode=register`
- 期望：`200`，返回 `user_id(uuid)`、`short_account`，并设置 Cookie

### TC-AUTH-005 重复注册失败
- 前置：手机号已注册
- 步骤：再次 `mode=register`
- 期望：`400`，提示手机号已注册

### TC-AUTH-006 未注册手机号登录失败
- 前置：手机号未注册
- 步骤：`POST /auth/verify`，`mode=login`
- 期望：`400`，提示先注册

### TC-AUTH-007 获取当前用户成功
- 前置：已登录
- 步骤：`GET /auth/me`
- 期望：`200`，`ok=true`，返回 `user.phone/user_id/short_account`

### TC-AUTH-008 获取当前用户失败（未登录）
- 步骤：不带 Cookie 调用 `GET /auth/me`
- 期望：`401`，`ok=false`，`message=未登录`

### TC-AUTH-009 退出登录
- 前置：已登录
- 步骤：`POST /auth/logout`
- 期望：`200`，Cookie 删除；再次访问 `/chat` 返回 `401`

## 3. 聊天接口

### TC-CHAT-001 未登录访问聊天
- 步骤：不带 Cookie 调用 `POST /chat`
- 期望：`401`，输出“请先登录后再继续聊天。”

### TC-CHAT-002 空 query
- 前置：已登录
- 步骤：`query` 为空或缺失
- 期望：`200`，返回引导文案“先告诉吉伊大师你想问什么”

### TC-CHAT-003 问候快捷回复
- 前置：已登录
- 步骤：`query=你好`
- 期望：命中快捷回复，不走复杂工具链

### TC-CHAT-004 时间快捷回复
- 前置：已登录
- 步骤：`query=今天几号`
- 期望：返回系统实时时间文案

### TC-CHAT-005 算命资料不足拦截
- 前置：已登录
- 步骤：`query=帮我算命`
- 期望：直接提示补姓名和出生年月日

### TC-CHAT-006 会话资料落库
- 前置：已登录
- 步骤：先发“我叫张三，2001-08-15”，再发“看事业运”
- 期望：第二次问题可复用资料，不重复追问

### TC-CHAT-007 session_id 入参兼容
- 前置：已登录
- 步骤：请求体传任意 `session_id`
- 期望：后端返回的 `session_id` 为用户 `uuid`（不依赖前端传值）

### TC-CHAT-008 异常兜底
- 前置：模拟外部 API 异常
- 步骤：发送复杂问题
- 期望：返回友好错误文案，不暴露堆栈

## 3.1 时间一致性回归（新增）

### TC-TIME-001 实时时间锚点
- 前置：已登录
- 步骤：`query=今天的日期`
- 期望：回复中的日期/星期与服务器当前时区时间一致

### TC-TIME-002 近几天窗口约束
- 前置：已登录，资料齐全（姓名+生日）
- 步骤：`query=近哪几天气场更顺`
- 期望：回复中出现的日期仅落在“今天起近3天”窗口内

### TC-TIME-003 日期纠错场景
- 前置：已登录；先发一条可能诱导旧日期的问题
- 步骤：再发 `query=你刚刚日期说错了，现在是几号`
- 期望：系统以当前时间锚点重答，不沿用上一轮旧年月

### TC-TIME-004 日期与星期匹配校验
- 前置：已登录
- 步骤：发送时间敏感问题（今天/近几天/本周）
- 期望：回复里若含“YYYY年M月D日 + 星期X”，必须匹配；不匹配时自动修正为锚点提示

## 4. 资料抽取

### TC-PROFILE-001 出生日期归一化
- 步骤：输入 `2005年6月15日`
- 期望：归一为 `2005-06-15`

### TC-PROFILE-002 姓名误识别过滤
- 步骤：输入“我是谁”
- 期望：不写入姓名字段

### TC-PROFILE-003 出生时刻提取
- 步骤：输入 `早上5:45`
- 期望：提取为 `05:45`

## 5. 知识入库

### TC-RAG-001 URL 入库成功
- 前置：URL 可访问
- 步骤：`POST /add_urls?URL=<有效地址>&force_recreate=false`
- 期望：返回“添加成功”，Qdrant 数据量增加

### TC-RAG-002 URL 入库失败场景
- 前置：URL 无效
- 步骤：调用 `/add_urls`
- 期望：接口报错或日志记录异常（按当前实现观察）

## 6. 前端联调

### TC-FE-001 登录页验证码流程
- 步骤：打开 `/login`，发送验证码、输入验证码登录
- 期望：提示文案正确，登录后跳转 `/index`

### TC-FE-002 注册页流程
- 步骤：打开 `/register`，发送验证码并注册
- 期望：注册成功后跳转 `/index`

### TC-FE-003 聊天消息渲染
- 步骤：在 `/index` 发送消息
- 期望：用户消息、思考态、机器人打字机效果依次出现

### TC-FE-004 退出登录
- 步骤：点击“退出登录”
- 期望：跳转 `/login`，再次访问 `/index` 被重定向

## 7. 回归建议
- 高优先级：`TC-AUTH-004`、`TC-AUTH-007`、`TC-CHAT-001`、`TC-CHAT-005`、`TC-CHAT-007`、`TC-FE-004`
- 每次改动鉴权、资料抽取、聊天流程后执行冒烟回归。

## 8. 命理质量回归组（P2）

### 8.1 覆盖目标
- 场景覆盖：`运势/桃花/财运/事业/学业/流年/占卜`
- 回归重点：命理路由命中、结构化依据完整、个人信息不回显、模板重复率可控

### 8.2 自动回归脚本
- 脚本：`scripts/fortune_regression.py`
- 默认执行 24 条用例（含缺资料追问、命理细问、占卜、通用问答）
- CI 工作流：`.github/workflows/fortune-quality-gate.yml`（自动执行回归并做指标门禁）

示例命令：

```bash
python3 scripts/fortune_regression.py --base-url http://127.0.0.1:8001
```

可选参数：
- `--max-cases 12`：仅跑前 12 条快速回归
- `--phone 13xxxxxxxxx`：指定测试手机号
- `--timeout 60`：请求超时秒数

### 8.3 质量指标接口
- 接口：`GET /quality/metrics?days=1`
- 统计维度（最近 N 天，N=1..7）：
  - `fortune_route_hit_rate`
  - `fortune_tool_success_rate`
  - `fortune_field_completeness_rate`
  - `profile_echo_violation_rate`
  - `template_repeat_rate`

### 8.4 发布门禁建议
- 命理意图路由命中率：`fortune_route_hit_rate >= 0.90`
- 命理工具成功率：`fortune_tool_success_rate >= 0.90`
- 字段完整率：`fortune_field_completeness_rate >= 0.85`
- 用户资料回显违规率：`profile_echo_violation_rate = 0`
- 模板重复率：`template_repeat_rate < 0.20`
