# 测试用例

- 文档版本：v2.0
- 更新日期：2026-02-27
- 适用版本：当前仓库主线

## 1. 测试目标与范围

目标：验证鉴权、资料融合、聊天路由、时间一致性、质量指标和知识入库链路可用。

范围：

- 鉴权与账号
- 聊天与命理路由
- 资料抽取与落库
- 前端关键交互
- 自动回归与质量门禁

## 2. 测试环境

### 2.1 基础前置

- 服务已启动（本地 8000 或 Docker 8001）
- MySQL、Redis 可用且完成初始化
- `.env` 已配置必要密钥
- 测试环境默认 `SMS_DEBUG_CODE_ENABLED=true`（便于自动化）

### 2.2 推荐环境矩阵

- E1：本地直连（`http://127.0.0.1:8000`）
- E2：Docker Compose（`http://127.0.0.1:8001`）

## 3. 功能测试用例

### 3.1 鉴权与账户

| 用例ID | 用例名称 | 步骤 | 预期结果 |
|---|---|---|---|
| TC-AUTH-001 | 发送验证码成功 | `POST /auth/send_code`，合法手机号，`scene=login` | `200`，`ok=true`，返回 `ttl_seconds=60` |
| TC-AUTH-002 | 发送验证码失败-手机号非法 | 手机号传 `12345` | `400`，提示手机号错误 |
| TC-AUTH-003 | 发送验证码失败-冷却中 | 60秒内重复发送 | 第二次 `429` |
| TC-AUTH-004 | 注册成功 | `mode=register` 且验证码正确 | `200`，返回 `user_id`、`short_account`，设置 Cookie |
| TC-AUTH-005 | 重复注册失败 | 已注册手机号再次注册 | `400` |
| TC-AUTH-006 | 未注册手机号登录失败 | `mode=login`，手机号未注册 | `400` |
| TC-AUTH-007 | 账号密码登录成功 | `POST /auth/login/password` | `200`，设置 Cookie |
| TC-AUTH-008 | 获取当前用户成功 | 已登录后 `GET /auth/me` | `200`，返回用户与资料 |
| TC-AUTH-009 | 获取当前用户失败 | 未登录调用 `GET /auth/me` | `401`，`ok=false` |
| TC-AUTH-010 | 退出登录 | 调用 `POST /auth/logout` | `200`，Cookie 删除 |
| TC-AUTH-011 | 忘记密码成功 | verify_code -> reset | `200`，可用新密码登录 |

## 4. 自动化回归

### 4.1 命理回归脚本

```bash
python3 scripts/fortune_regression.py --base-url http://127.0.0.1:8001 --max-cases 24
```

默认覆盖：命理、决策、口语时窗、星座澄清、解梦、占卜、通用场景。

### 4.2 质量门禁脚本

```bash
python3 scripts/quality_gate.py --base-url http://127.0.0.1:8001 --days 1
```

默认阈值关注：

- `fortune_route_hit_rate >= 0.90`
- `fortune_tool_success_rate >= 0.90`
- `fortune_field_completeness_rate >= 0.85`
- `profile_echo_violation_rate <= 0.0`
- `unique_output_rate >= 0.45`
- `max_pair_similarity <= 0.92`
- `observability_coverage >= 0.99`

## 5. 回归执行建议

- 冒烟必跑：`TC-AUTH-004`、`TC-AUTH-008`、`TC-CHAT-001`、`TC-CHAT-005`、`TC-TIME-001`
- 修改路由策略后：全量运行自动化回归 + 门禁
- 发布前：至少在 E1/E2 两套环境各跑一轮核心用例
