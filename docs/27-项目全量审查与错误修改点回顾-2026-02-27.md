## 3. 本轮联调新增问题与修复（运行侧）

## 3.1 P0：清库后注册/登录 500（`/auth/verify`）

- 现象：前端注册失败，控制台报 `POST /auth/verify 500`。
- 实际报错：`Table 'fortune_telling.users' doesn't exist`。
- 根因：清空 volume 后，MySQL 没有自动建表。

### 修复

- 在 Compose 中挂载初始化 SQL：
  - `docker-compose.yml:52`
  - `./sql/001_init_auth_schema.sql:/docker-entrypoint-initdb.d/001_init_auth_schema.sql:ro`

### 验证

- 重建后 `mysql` healthy，`/auth/send_code`、`/auth/verify` 恢复 200。

## 3.2 P0：fortune 早返回分支未写会话历史

- 现象：日志里 `POST /chat 200`，但下一轮“记忆断裂”，出现“问过但忘记”的体验。
- 根因：`fast_reply / dream / zodiac / fortune` 分支早返回前未写入 `RedisChatMessageHistory`。

### 修复

- 新增 `_append_chat_history(...)`：`server.py:1514`
- 在以下分支补写会话：
  - `fast_reply`：`server.py:3203`
  - `dream`：`server.py:3223`
  - `zodiac`：`server.py:3250`
  - `fortune`：`server.py:3296`

### 验证

- Redis `message_store:<uuid>` 列表长度随对话增加；可检索到 fortune 分支的 human/ai 成对记录。

## 3.3 P1：裸资料输入未识别姓名，导致误追问姓名

- 场景：用户输入 `刘芷华 2005-06-15 早上5:45` 后再问爱情运势。
- 现象：系统可能仍提示“请告诉我你的姓名”。
- 根因：资料提取规则仅覆盖 `我叫/我是/姓名是`，未覆盖“姓名+生日”裸输入。

### 修复

- 扩展 `extract_profile_from_query(...)` 裸输入匹配：`server.py:1451-1458`

### 验证

- `user_profile` 可写入姓名与生日时间。
- 后续 `查询2026年的爱情运势` 可直接返回，不再强制补姓名。
