# 数据库设计说明书

- 文档版本：v2.0
- 更新日期：2026-02-27
- 依据文件：`sql/001_init_auth_schema.sql`、`server.py`

## 1. 数据存储总览

系统采用三层存储：

- MySQL：核心持久化数据
- Redis：高频状态与短期数据
- Qdrant（本地）：向量检索数据

## 2. MySQL 设计

### 2.1 表清单

- `users`：用户主表
- `user_profile`：用户资料
- `auth_sessions`：登录会话审计
- `chat_messages`：聊天消息（预留，当前主链路未写入）
- `sms_code_logs`：短信审计（预留）
- `password_reset_logs`：密码重置审计

### 2.2 表结构说明

#### 2.2.1 `users`

关键字段：

- `id` BIGINT PK
- `uuid` CHAR(32) UNIQUE
- `account` VARCHAR(24) UNIQUE，格式 `JIYI-XXXXXXXX`
- `phone` VARCHAR(20) UNIQUE
- `password_hash` VARCHAR(255)（`pbkdf2_sha256$...`）
- `status` TINYINT
- `created_at` / `updated_at`

代码使用：

- 查询：`_get_user_by_phone` / `_get_user_by_account` / `_get_user_by_uuid`
- 创建：`_create_user_by_phone`
- 更新密码：`_update_user_password`

#### 2.2.2 `user_profile`

关键字段：

- `user_id` BIGINT PK, FK -> `users.id`
- `name` VARCHAR(64)
- `birth_date` DATE
- `birth_time` TIME
- `profile_json` JSON（保存 `preferred_name` 等扩展资料）
- `updated_at`

代码使用：

- 查询：`_get_profile_by_user_id`
- 更新：`_merge_profile_to_db`

更新策略：

- 姓名/生日/时辰仅补齐不覆盖
- `preferred_name` 支持更新为最新偏好

#### 2.2.3 `auth_sessions`

关键字段：

- `id` BIGINT PK
- `user_id` BIGINT FK
- `token_hash` CHAR(64) UNIQUE（SHA-256）
- `login_type` ENUM(`sms`,`password`)
- `device_info` / `ip`
- `expires_at` / `created_at` / `revoked_at`

代码使用：

- 写入：`_save_auth_session_to_db`
- 撤销：`_revoke_auth_session_in_db`

#### 2.2.4 `password_reset_logs`

关键字段：

- `id` BIGINT PK
- `user_id`
- `phone`
- `reset_at`
- `ip`
- `user_agent`

代码使用：`_log_password_reset`

### 2.3 索引与约束

- 唯一索引：`users.uuid`、`users.account`、`users.phone`、`auth_sessions.token_hash`
- 常用索引：
  - `idx_auth_user_created`
  - `idx_auth_expires`
  - `idx_reset_user_time`
- 外键：`user_profile.user_id`、`auth_sessions.user_id`

## 3. Redis 设计

### 3.1 Key 模式与 TTL

| 用途 | Key 模式 | TTL | 说明 |
|---|---|---|---|
| 登录态 | `auth:session:{token}` | 30天 | 存 `phone`、`user_uuid` |
| 验证码 | `auth:sms:{scene}:{phone}` | 300秒 | 场景支持 `login/register/reset_password/default` |
| 验证码冷却 | `auth:sms:cooldown:{scene}:{phone}` | 60秒 | 发送频率控制 |
| 重置密码校验态 | `auth:pwdreset:verified:{phone}` | 600秒 | 忘记密码流程中间态 |
| 聊天历史 | `RedisChatMessageHistory(session_id=user_uuid)` | `SESSION_TTL_SECONDS` | 默认600秒 |
| 偏好追问状态 | `chat:preferred_name_prompt:{session_id}` | 24小时 | 控制“称呼偏好”追问 |
| 质量指标 | `jiyi:quality:*` | 14天 | 看板与门禁统计 |
| 特性开关 | `jiyi:feature_flags:v2` | 手工维护 | 灰度与回滚 |

## 4. Qdrant 设计

- 路径：`VECTOR_DB_PATH`（默认 `./local_qdrand`）
- 集合：`VECTOR_COLLECTION_NAME`（默认 `local_documents`）
- 写入入口：`POST /add_urls`
- 检索入口：工具函数 `get_info_from_local_db`

## 5. 一致性与事务策略

- 账户与资料：以 MySQL 为最终一致性源（SoT）
- 会话与验证码：以 Redis 实时状态为准
- 聊天历史：当前为短期会话上下文，不作为长期审计数据
- 登录流程采用“Redis登录态 + MySQL审计”双写，容忍审计轻微延迟

## 6. 初始化与运维

### 6.1 初始化

- Docker 模式自动加载：`sql/001_init_auth_schema.sql`
- 本地模式需提前建库并执行同名 SQL

### 6.2 运维建议

- 为 MySQL 启用定期备份（至少每日）
- 为 Redis 配置持久化策略（RDB/AOF）按业务要求选型
- 定期清理历史审计数据并归档

## 7. 安全注意事项

- 密码仅存哈希，不存明文
- 审计表存 token 哈希，不存原 token
- 生产环境建议开启数据库最小权限账号
