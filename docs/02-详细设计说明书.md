# 详细设计说明书

## 1. 范围
本文档基于当前代码实现（`server.py`、`mytools.py`、`models.py`、`static/js/*.js`）描述详细设计，不包含未实现功能。

## 2. 后端详细设计

## 2.1 配置与初始化
- 配置来源：`config.py`（`.env`）
- 关键配置项：`REDIS_URL`、`MYSQL_*`、`SESSION_TTL_SECONDS`、`VECTOR_*`、`SERPAPI_API_KEY`、`YUANFENJU_API_KEY`
- `server.py` 启动后：
  - 初始化 Redis 客户端 `_REDIS_CLIENT`
  - 定义 MySQL 连接函数 `_db_conn()`
  - 挂载静态目录 `/static`
  - 加载模板目录 `templates/`

## 2.2 存储访问层（server.py 内部函数）

### 2.2.1 MySQL 访问
- `_get_user_by_phone(phone)`：按手机号查用户
- `_get_user_by_uuid(user_uuid)`：按 UUID 查用户
- `_create_user_by_phone(phone)`：创建用户并初始化 `user_profile`
- `_get_profile_by_user_id(user_id)`：读资料（name/birth_date/birth_time）
- `_merge_profile_to_db(user_id, current)`：仅补齐缺失资料，不覆盖已有资料
- `_save_auth_session_to_db(...)`：记录登录会话审计
- `_revoke_auth_session_in_db(token)`：登出时标记会话撤销

### 2.2.2 Redis 访问
- `_set_auth_session/_get_auth_session/_delete_auth_session`
  - Key：`auth:session:{token}`
- `_set_sms_code/_get_sms_code/_delete_sms_code/_sms_cooldown_ttl`
  - Key：`auth:sms:{scene}:{phone}`
  - 冷却 Key：`auth:sms:cooldown:{scene}:{phone}`

## 2.3 聊天编排核心（Master）

### 2.3.1 初始化
- LLM 客户端：`get_lc_ali_model_client()`
- Prompt 结构：
  - System（角色+规则）
  - 历史消息占位（`chat_history`）
  - Human（当前问题）
  - `agent_scratchpad`
- Memory：`ConversationBufferMemory` + `RedisChatMessageHistory`
- Tools：`serp_search`、`get_info_from_local_db`、`bazi_cesuan`、`yaoyigua`、`jiemeng`

### 2.3.2 执行
- `emotion_chain(query)`：先判定情绪标签
- `run(query)`：执行 agent，异常时返回兜底文案

## 2.4 资料抽取与消息处理
- `extract_profile_from_query`：从用户输入抽取姓名/生日/时间
- `extract_profile_from_history`：从 Redis 历史消息抽取资料
- `merge_session_profile(session_id, current)`：通过 `uuid` 查用户并写 MySQL
- `build_profile_context`：将已知资料拼接为模型可用上下文
- `sanitize_output`：清洗模型输出（去符号、去重复、去固定无效话术）
- `get_fast_reply`：时间/问候类问题快速回复

## 2.5 路由详细设计

### 2.5.1 页面路由
- `GET /`、`GET /index`
  - 读取 Cookie -> Redis 登录态
  - 未登录 302 到 `/login`
  - 已登录渲染 `index.html`（注入 `account/uuid`）
- `GET /login`、`GET /register`
  - 已登录则 302 到 `/index`

### 2.5.2 账户接口
- `GET /auth/me`
  - 基于 Cookie 返回当前用户与资料
- `POST /auth/send_code`
  - 入参：`phone`、`scene`
  - 校验手机号 + 冷却（`429`）
  - 生成 6 位验证码写 Redis
  - 返回 `ttl_seconds`；开发/演示环境可附带 `debug_code`
- `POST /auth/verify`
  - 入参：`phone`、`code`、`mode(login/register)`
  - 校验验证码（先按 scene，再回退 default）
  - `mode=register` 时要求 `password`（8-12位字母或数字）
  - 注册时创建用户；登录时校验用户存在
  - 写 Redis 会话 + MySQL 审计，设置登录 Cookie
- `POST /auth/login/password`
  - 入参：`account`、`password`
  - 校验账号密码并写会话
- `POST /auth/password/verify_code`
  - 忘记密码流程校验短信验证码
- `POST /auth/password/reset`
  - 校验通过后重置密码并记录审计
- `POST /auth/logout`
  - 删除 Redis 会话
  - 更新 MySQL 审计撤销时间
  - 删除 Cookie

### 2.5.3 聊天接口
- `POST /chat`
  - 未登录返回 `401`
  - 空 query 返回引导文案
  - `session_id` 强制绑定为用户 `uuid`
  - 先补资料，再判定是否资料不足（算命场景）
  - 命理/占卜场景优先走专用路由；其余场景再走 Agent
  - 返回 `{session_id, output}`

### 2.5.4 运维接口
- `GET /quality/metrics`
  - 入参：`days=1..7`
  - 返回命理质量指标聚合数据

### 2.5.5 知识接口
- `POST /add_urls`
  - 入参：`URL`、`force_recreate`
  - 抓取页面 -> 切分 -> 向量化 -> 写 Qdrant

## 2.6 前端详细设计

## 2.6.1 登录注册页
- 输入校验：11 位手机号、6 位验证码
- 登录页脚本：`login_multi.js`（短信登录 + 账号密码登录）
- 注册页脚本：`auth.js`（手机号+验证码+密码注册）
- 发送验证码：`/auth/send_code`
- 校验登录/注册：`/auth/verify` 或 `/auth/login/password`
- 忘记密码页脚本：`reset_password.js`（`/auth/password/verify_code` + `/auth/password/reset`）
- 成功跳转 `/index`

## 2.6.2 聊天页（main.js）
- 前端不再主导会话 ID（后端按 UUID 绑定）
- 页面初始化调用 `/auth/me` 预加载资料
- 聊天请求统一调用 `/chat`
- UI：思考态、打字机效果、建议按钮、登出

## 3. 异常处理与返回策略
- JSON 解析失败：`400`
- 参数格式错误（手机号/验证码）：`400`
- 发送过快：`429`
- 未登录：`401`
- Agent/第三方异常：返回友好文本，不向前端暴露堆栈
