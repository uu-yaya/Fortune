## 2. 初始化与配置

### 2.1 配置来源

- 文件：`config.py`
- 加载方式：`python-dotenv` 读取 `.env`
- 关键配置：
  - Redis：`REDIS_URL`、`SESSION_TTL_SECONDS`
  - MySQL：`MYSQL_HOST` / `MYSQL_PORT` / `MYSQL_DB` / `MYSQL_USER` / `MYSQL_PASSWORD`
  - 向量库：`VECTOR_DB_PATH`、`VECTOR_COLLECTION_NAME`
  - 外部能力：`DASHSCOPE_API_KEY`、`SERPAPI_API_KEY`、`YUANFENJU_API_KEY`

### 2.2 应用启动

`server.py` 启动时完成：

- FastAPI 实例初始化
- Redis 客户端 `_REDIS_CLIENT` 初始化
- 静态目录 `/static` 与模板目录 `templates/` 挂载
- LLM / Embedding 与工具链按需加载

## 3. 数据访问层设计

### 3.1 MySQL 访问函数

- 用户
  - `_get_user_by_phone`
  - `_get_user_by_account`
  - `_get_user_by_uuid`
  - `_create_user_by_phone`
  - `_update_user_password`
- 用户资料
  - `_get_profile_by_user_id`
  - `_merge_profile_to_db`
- 会话审计
  - `_save_auth_session_to_db`
  - `_revoke_auth_session_in_db`
- 密码重置日志
  - `_log_password_reset`

设计要点：

- 密码采用 `pbkdf2_sha256`（200000 轮）
- 资料更新为“仅补齐缺失字段 + 可更新称呼偏好”
- 会话审计按 token 哈希落库

### 3.2 Redis 访问函数

- 鉴权会话：`auth:session:{token}`
- 验证码：`auth:sms:{scene}:{phone}`
- 冷却计时：`auth:sms:cooldown:{scene}:{phone}`
- 密码重置校验态：`auth:pwdreset:verified:{phone}`
- 会话风格态：`jiyi:reply_style:{session_id}`
- 质量指标：`jiyi:quality:*`

## 5. 路由与业务设计

### 5.1 页面路由

- `GET /`、`GET /index`：已登录渲染聊天页，否则重定向 `/login`
- `GET /login`、`GET /register`：已登录重定向 `/index`
- `GET /forgot-password`、`GET /reset-password`：忘记密码页

### 5.2 鉴权路由

- `POST /auth/send_code`
  - 校验手机号 + 冷却
  - 生成验证码写 Redis，开发模式可返回 `debug_code`
- `POST /auth/verify`
  - 支持 `mode=login/register`
  - 注册时校验密码规则并建用户
  - 写登录态 Cookie + MySQL 审计
- `POST /auth/login/password`
  - 按账号密码登录
- `POST /auth/password/verify_code`
  - 校验忘记密码验证码并写短期校验态
- `POST /auth/password/reset`
  - 校验通过后更新密码并落审计日志
- `GET /auth/me`
  - 返回当前用户和资料（含 `preferred_name`）
- `POST /auth/logout`
  - 清理 Redis 登录态、撤销审计会话、删除 Cookie

### 5.3 聊天路由 `POST /chat`

主链路流程：

1. 鉴权校验
2. 时间锚点构建（`build_time_anchor`）
3. 会话 ID 绑定用户 UUID
4. 资料融合：
   - `extract_profile_from_history`
   - `extract_profile_from_query`
   - `merge_session_profile`
5. 意图识别与路由：
   - `get_fast_reply`
   - `route_dream_pipeline`
   - `route_zodiac_pipeline`
   - `route_fortune_pipeline`
   - `Master.run`（Agent 回退）
6. 输出后处理：
   - `sanitize_output`
   - `strip_profile_echo`
   - `validate_time_consistency`
7. 指标上报：`track_output_quality` + `_log_route_observability`

### 5.4 质量与知识路由

- `GET /quality/metrics`：汇总近 1~7 天质量指标
- `POST /add_urls`：抓取 URL 并入库 Qdrant

## 6. 命理/时间/质量详细策略

### 6.1 命理路由策略

- 非命理问题不进入命理管线
- 命理问题必须先满足资料完整（姓名 + 出生日期）
- 命理工具失败时返回可解释降级内容，不抛出堆栈

### 6.2 时间一致性策略

- 对时间敏感问句执行日期/星期一致性校验
- 支持窗口修补与严重冲突回退
- 关键指标：
  - `temporal_consistency_hit_rate`
  - `time_validation_fail_rate`
  - `time_validation_autofix_rate`

### 6.3 质量指标设计

`get_quality_metrics` 聚合主要指标：

- 命理链路：`fortune_route_hit_rate`、`fortune_tool_success_rate`
- 输出质量：`template_repeat_rate`、`unique_output_rate`、`max_pair_similarity`
- 安全合规：`profile_echo_violation_rate`
- 可观测性：`observability_coverage`

## 7. 前端详细设计

### 7.1 登录与注册页

- 文件：`static/js/login_multi.js`、`static/js/auth.js`
- 功能：短信验证码登录、账号密码登录、注册流程
- 交互：发送验证码倒计时、参数前置校验、错误提示

### 7.2 忘记密码页

- 文件：`static/js/reset_password.js`
- 流程：发送验证码 -> 校验验证码 -> 设置新密码

### 7.3 聊天页

- 文件：`static/js/main.js`
- 行为：
  - 初始化拉取 `/auth/me`
  - 发送消息到 `/chat`
  - 打字机渲染、建议按钮、登出

## 8. 异常与返回策略

- 参数错误：`400`
- 未登录：`401`
- 发送过快：`429`
- 业务异常：返回友好文案 + 错误编号（不暴露堆栈）

## 9. 现阶段边界

- 验证码回传仅适用于开发调试。
- 会话历史以 Redis 短期保存为主。
- 外部模型/工具依赖波动会影响回答质量上限。
