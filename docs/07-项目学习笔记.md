## 3. 后端深读（server.py）

## 3.1 配置与基础依赖
- `AUTH_COOKIE_NAME = jiyi_auth_token`
- 验证码 TTL：300 秒，重发冷却：60 秒，登录态 TTL：30 天。
- Redis 客户端初始化：`redis.Redis.from_url(REDIS_URL, decode_responses=True)`。
- MySQL 连接函数：`_db_conn()`（pymysql + DictCursor + autocommit）。

## 3.2 鉴权与会话存储
- Redis Key：
  - `auth:session:{token}`：登录态 JSON。
  - `auth:sms:{scene}:{phone}`：验证码。
  - `auth:sms:cooldown:{scene}:{phone}`：重发冷却。
- MySQL 审计：
  - `auth_sessions` 写入 `_save_auth_session_to_db()`。
  - 登出时 `_revoke_auth_session_in_db()`。

## 3.3 用户与资料
- 用户查询：`_get_user_by_phone()` / `_get_user_by_uuid()`。
- 新建用户：`_create_user_by_phone()`。
  - 自动生成 `uuid` 与 `account=JIYI-XXXXXXXX`。
  - 同时创建 `user_profile` 空记录。
- 资料融合：`_merge_profile_to_db()`。
  - 仅“补齐缺失字段”，不会覆盖已存在数据。
  - 字段：`name`、`birth_date`、`birth_time`。

## 3.4 聊天核心链路
1. `POST /chat` 校验 Cookie -> Redis 登录态。
2. 用 phone 查 MySQL 用户，`session_id` 绑定用户 `uuid`。
3. 用 `RedisChatMessageHistory` 读取短期历史。
4. `extract_profile_from_history` + `extract_profile_from_query` 抽取资料。
5. `merge_session_profile` 把资料落 MySQL。
6. 运势关键词命中且姓名/生日缺失时，直接提示补资料。
7. 命中 `get_fast_reply`（时间/问候）则快速返回。
8. 其余走 `Master.run` 调用 LangChain Agent + tools。
9. `sanitize_output` 清洗后返回。

## 3.5 路由一览
- 页面：`GET /`、`GET /index`、`GET /login`、`GET /register`
- 鉴权：`GET /auth/me`、`POST /auth/send_code`、`POST /auth/verify`、`POST /auth/logout`
- 业务：`POST /chat`、`POST /add_urls`

## 9. 常见故障定位路径
1. 登录失败：先看 `/auth/send_code` 是否 429（冷却中）或 400（手机号/验证码格式）。
2. 登录后跳回登录页：检查 Redis `auth:session:{token}` 是否存在。
3. 聊天 401：Cookie 丢失或 Redis session 过期。
4. 聊天无法保持资料：检查 MySQL `user_profile` 更新权限与字段类型。
5. 工具无结果：分别确认对应 API key；`mytools.py` 有降级文案。
