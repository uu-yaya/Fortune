## 2. 推荐阅读顺序

新成员建议按以下顺序建立认知：

1. `README.md`（仓库总览）
2. `docs/01` -> `docs/05`（设计、接口、测试基线）
3. `docs/12` + `docs/14`（V2问题与方案）
4. `docs/13` + `docs/15~26`（测试与改造结果）
5. `docs/27`（复盘与问题闭环）

## 3. 维护规则

- 规范基线文档（01~05）必须与代码保持同步更新。
- 测试报告必须包含：执行时间、环境、输入范围、指标与结论。
- 新增阶段报告建议按“序号-主题-日期”命名，避免同名覆盖。
- 若结论进入长期规范，请回写到 01~05，而不是只留在阶段报告。
