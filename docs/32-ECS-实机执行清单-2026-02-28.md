# 32-ECS 实机执行清单-2026-02-28

> 目标：在阿里云 ECS 上完成 password-only 首发（账号密码登录 + 聊天），禁用短信/注册/找回密码入口。  
> 说明：以下命令按顺序执行，遇到占位符请先替换。

## 0. 准备变量（在本地终端）

```bash
ECS_IP="你的ECS公网IP"
DOMAIN="你的域名"
REPO_URL="你的仓库地址"
```

## 1. 登录 ECS

```bash
ssh root@${ECS_IP}
```

## 2. 安装 Docker / Compose / Nginx / Certbot

```bash
apt-get update
apt-get install -y ca-certificates curl git gnupg lsb-release nginx certbot python3-certbot-nginx

install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /etc/apt/keyrings/docker.gpg
chmod a+r /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" > /etc/apt/sources.list.d/docker.list
apt-get update
apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

systemctl enable docker
systemctl start docker
docker --version
docker compose version
```

## 3. 拉取项目代码

```bash
mkdir -p /opt/fortune-telling
cd /opt/fortune-telling
git clone ${REPO_URL} .
```

## 4. 准备生产 `.env`（password-only）

```bash
cp deploy/env/.env.password-only.example .env
vim .env
```

至少确认以下配置：

- `SMS_PROVIDER=mock`
- `SMS_DEBUG_CODE_ENABLED=false`
- `DASHSCOPE_API_KEY=你的真实值`
- `MYSQL_ROOT_PASSWORD=强密码`

## 5. 启动应用容器

```bash
docker compose up -d --build
docker compose ps
docker compose logs --tail=120 numerology
```

## 6. 申请 HTTPS 证书（Let’s Encrypt）

```bash
systemctl stop nginx
certbot certonly --standalone -d ${DOMAIN}
systemctl start nginx
```

## 7. 部署 Nginx password-only 网关配置

```bash
cp deploy/nginx/password-only.conf.example /etc/nginx/conf.d/fortune-telling.conf
sed -i "s/example.com/${DOMAIN}/g" /etc/nginx/conf.d/fortune-telling.conf
sed -i "s#/etc/nginx/ssl/${DOMAIN}/fullchain.pem#/etc/letsencrypt/live/${DOMAIN}/fullchain.pem#g" /etc/nginx/conf.d/fortune-telling.conf
sed -i "s#/etc/nginx/ssl/${DOMAIN}/privkey.pem#/etc/letsencrypt/live/${DOMAIN}/privkey.pem#g" /etc/nginx/conf.d/fortune-telling.conf

nginx -t
systemctl reload nginx
```

## 8. 预置首批账号（无短信）

```bash
docker compose exec -T numerology python scripts/bootstrap_password_only_accounts.py \
  --entry 13800138000:TempA123 \
  --entry 13900139000:TempB123
```

记录输出的 `phone -> account` 映射，并通过安全渠道发放 `account/password`。

## 9. 上线验收

```bash
BASE_URL="https://${DOMAIN}" \
ACCOUNT="刚创建出来的账号" \
PASSWORD="TempA123" \
bash scripts/password_only_acceptance.sh
```

## 10. 运营期常用命令

查看状态与日志：

```bash
docker compose ps
docker compose logs --tail=200 numerology
```

重置密码（工单场景）：

```bash
docker compose exec -T numerology python scripts/reset_user_password.py \
  --account JIYI-XXXXXXXX \
  --new-password NewPass88
```

## 11. 快速回滚（恢复原入口）

1. 编辑 `/etc/nginx/conf.d/fortune-telling.conf`，移除短信接口 `410` 拦截与注册/找回 `302` 跳转规则。  
2. 执行：

```bash
nginx -t
systemctl reload nginx
docker compose up -d
```
